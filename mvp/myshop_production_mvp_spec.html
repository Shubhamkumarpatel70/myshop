Production architecture layers
1

Client layer — React PWA

Single React app with 3 separate route guards: /admin, /owner, /pos. PWA manifest + service worker for offline POS queue. Framer Motion only on non-POS screens (perf budget).

2

API gateway — Express + rate limiting

express-rate-limit on all auth routes (5 req/min). Helmet.js headers. CORS whitelist. All routes behind verifyToken middleware. Role checked at route level, not just client.

3

Auth — JWT + refresh token

Access token: 15 min expiry. Refresh token: 7 days, stored in httpOnly cookie (not localStorage). Silent refresh via /auth/refresh endpoint. Refresh token rotation on each use.

4

Business logic — service layer

Controllers stay thin. Business logic (health score calc, invoice numbering, stock deduction) lives in /services. Makes unit testing possible without HTTP overhead.

5

Database — MongoDB Atlas

Indexes on: memberId, shopId, productBarcode, createdAt. Use transactions for POS checkout (stock deduct + sale create must be atomic). Connection pool via Mongoose.

6

File storage — Cloudinary / S3

All KYC images, payment screenshots go to cloud storage on upload. Store only the URL in MongoDB. Signed URLs for admin access. Delete local Multer storage entirely.

7

Background jobs — node-cron

Daily: low-stock alert emails. Daily: expiry warning (items expiring in 7 days). Weekly: health score recalculation. These must NOT run in the request lifecycle.

All 8 production modules
Module 1

Onboarding & KYC

Registration → Aadhar upload → UPI payment → Status tracker (live stepper) → Admin review → Auto dashboard unlock via socket or polling. KYC rejection must include reason + resubmit option.

Module 2

Auth & role management

4 roles: super_admin, shop_owner, manager, cashier. Owner creates staff, assigns role. Each role has a permission matrix enforced server-side. Staff PIN login for fast POS switch.

Module 3

POS billing

Barcode scan + manual name search. Cart with qty edit. GST auto-apply by category. Cash / UPI / split tender. Print + WhatsApp share invoice. Offline queue with sync indicator. Return sale flow.

Module 4

Inventory engine

Add / edit / delete products. Bulk CSV import. Barcode generation. Low-stock threshold per item. Expiry date tracking with 7-day warning. Category management. Stock adjustment log.

Module 5

Business intelligence

Health Score with breakdown (inventory fill %, expiry risk %, revenue trend). Daily / monthly sales report. Top 10 products. Revenue by category chart. PDF export. All filterable by date range.

Module 6

Shift management

Open shift → close shift. Expected cash vs actual cash reconciliation. Variance alert if > ₹100 difference. Shift summary: total sales, total items sold, top cashier. Manager can view all shifts.

Module 7

Admin control centre

KYC review queue with bulk approve/reject. Rejection reason required. Payment verification (screenshot + UPI amount check). Shop search by member ID. Audit log. Global settings: fee, UPI ID, announcements.

Module 8

Notifications & alerts

In-app: approval status, low stock, expiry. Email: daily sales summary, shift close report. WhatsApp (V2 ready): low stock, invoice share. Admin broadcast to all/specific shops. Notification preferences per user.

Responsiveness rules

Breakpoints

360px: POS-only layout, single column, large tap targets. 768px: owner dashboard, sidebar collapses to bottom nav. 1024px+: full admin layout, sidebar fixed.

POS tap targets

Every button min 48×48px. Product grid: 2 columns on phone, 4 on tablet. Quantity +/- buttons at least 44px. Checkout button: full width, 56px tall.

Performance budget

Remove backdrop-filter blur from cards and sidebar on screens under 768px. Framer Motion disabled on POS route entirely. Target: 60fps on Redmi 9A (budget benchmark device).

Dark mode

Read prefers-color-scheme on first load. Set class on html element via blocking inline script to prevent flash of wrong theme. Store preference in localStorage.

Form & error UX rules

Inline validation

No browser alert(). No toast-only errors for form fields. Red border + error message below each invalid field on blur. Green checkmark on valid.

Loading states

Every API call shows a skeleton or spinner in the exact space the data will occupy. No blank screens. Inventory list, dashboard tiles, and reports all need skeleton loaders.

Optimistic UI on POS

When adding item to cart, update cart count instantly. Revert on API error. Don't make cashier wait for a server round-trip to see the cart update.

Empty states

Every list (inventory, sales, staff) needs an empty state illustration + action button ("Add your first product"). No blank table with just column headers.

Security requirements
Critical

Server-side role enforcement

Never trust client-side role. Every protected route calls hasPermission(user.role, 'action') middleware. Frontend hiding is UX, not security.

Critical

Input sanitization

Use express-validator on all POST/PUT routes. Mongoose schema types prevent most injection, but validate types, lengths, and formats at route level too.

High

Rate limiting

Auth routes: 5 attempts / minute. OTP routes: 3 attempts / 10 minutes. General API: 100 req / minute per IP. Use express-rate-limit + Redis store in production.

High

File upload validation

MIME type check (not just extension). Max size: 5MB for images. Use sharp to re-encode images before storing — strips EXIF and prevents polyglot attacks.

Standard

Helmet.js headers

Content-Security-Policy, X-Frame-Options, X-Content-Type-Options, HSTS. Configure in Express before any route definition.

Standard

Environment variables

Zero secrets in code. .env for local, environment config in hosting platform (Railway / Render / EC2). Rotate keys on any team member departure.

Infrastructure decisions

Hosting

Backend: Railway or Render (free tier to start, auto-deploy from Git). Frontend: Vercel. Both have zero-downtime deploys.

Database

MongoDB Atlas M0 (free) to start. Enable IP whitelist. Daily automated backups. Upgrade to M10 at ~50 shops.

File storage

Cloudinary free tier: 25GB storage, 25GB bandwidth. Enough for 500+ shops' KYC docs. Use signed upload presets.

Email

Resend (free tier: 3,000/month). For daily summaries and KYC notifications. Use React Email for templates.

Monitoring

Sentry (free tier) for error tracking. UptimeRobot for uptime pings every 5 min. Console.log is not monitoring.

CI/CD

GitHub Actions: lint + test on PR, auto-deploy main to staging, manual promote to production. Never deploy untested code.
